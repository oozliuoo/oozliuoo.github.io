Simple OS Definition:

  • A special piece of software that abstracts and arbitrates the underlying hardware
    • Abstract: simplify
    • Arbitrate: manage, oversee and control

Visual Metaphor: Operating System v.s. Toy shop Manager

Features/Functionalities Operating System Toy shop Manager
Direct operational resources Control use of CPU, Memory control use of employee time, parts & tools
Enforce working policies Fair resource access, limits to resource usage Fairness, safety, cleanup
Migrates difficulty of computer tasks Abstract hardware details (system call) Simplifies operation & optimize performance

What is an Operating System

  • Hide hardware complexity.
  • Resource Management
  • Provide isolation & protection
  • Directly has priviledged access to underlying hardware

Operating System Elements

  1. Abstractions
    • software
      • process, thread
    • hardware
      • file
      • socket
      • memory page

  2. Mechanisms

    • create, schedule
    • open, write, allocate

  3. Policies

    • Least-recently used (LRU)
    • Earliest deadline first (EDF)
      â€¦

Design Principles

  • Separation of mechanisms & policies
  • Optimize for common case

Basic Operating System Services

  • Scheduler -> CPU
  • Main Memory (Mm) Manager -> Main Memory
  • Block Device Driver
  • File System

Different Types of Operating Systems

  1. Monolithic Operating System

    • Pros:
      • Compile time optimizations
    • Cons:
      • All services and abstractions are already in an Operating System
      • Really large
      • Hard to maintain

  2. Modular Operating System

    • Everything can be added as modules
    • Operating System contains interfaces to allow alternatives
    • Pros:
      • Smaller code basis
      • Less resource needed
      • Easier to maintain
    • Cons:
      • Indirection impact on performance

  3. Microkernel:

    • Only require the most basic services. e.g. Address Space, threads
    • All others reside in user level
    • Requires lots of inter process communication
    • Pros:
      • Very small
    • Cons:
      • Portability will be bad.
      • Complexity of Software Development
      • Cost of user/kernel crossing
