NanoDegree AI - Simulated Annealing

Posted on 2017-04-14

Introduction

The philosophy behind this is to start from simple, and add a little intelligence only when necessary, like iteratively climbing up a hill. However, there could be local maximum/minimum that we need to avoid

Typical Example

n-Queen problems: moving a queen to reduce conflicts(attacks) is a simulation of hill climbing, and when there is no move to reduce conflicts yet the puzzle is not solve, that is a local maximum

Hill Climbing and Local Maximum

Solution

Do a random restart. It is not very efficient, as we need to do a lot of random restart to collect examples and end up with a good result

Improvements

  1. A Taboo Search would be more efficient, which will keep track of spaces you have been (restarted), and only restart at unknown positions
  2. A Variant of Stage Algorithm would also perform better, which will keep a list of all local maximum, and get a general shape of the problem

Other Problems

  • If step size is too small, it might be stuck at the shoulder, and cannot see the upcoming increase. Need to determine when (i.e. how many tries on 0 improvements) to stop
  • If step size is too large, it could possibly skip the maximum, and could sometimes get to infinit loop situation

Simulated Annealing

Annealing

Heating to hight temperature to increase the randomness, and then gradually cool it down to converge to a solution

Local Beam search

  • Instead of starting at only one position while performing Simulated Annealing, start at k positions, and keep the k best positions
  • If any of these k best positions reach the goal, we terminate

Genetic Algorithm

An analogy of the natural selection in biology, and uses breeding and mutation to find the optimal solution to a problem

General procedure

  1. Evaluate all randomly selected states (chosen as parents), assign a score to each of them
  2. Based on the score, normalize and calculate the percentage such that the related state will be selected to “breed” children
  3. Select another set of parents based on the percentages calculated in step 2)
  4. Do crossover and mutation to breed children using pairs of the parents
  5. Using the new generations as parents and repeat step 1, till we have a good generation that could optimally fix the problem

Cross-over

Cross over simply means crossing the parents randomly, and use part of them to combine with the other parent, and generate the next generation

Mutation

  • Motivation: critical part or information might be lost or never reached in some cases, so we want to introduce more randomness to bring them back. Just like mutation in biology
  • Each piece of information of the state will have a small but significant chance to be mutated to another piece of information
