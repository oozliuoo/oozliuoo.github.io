Type of Constraint

Unary Constraint

  • Restrict the value of a single variable.
  • Example, v1 != value

Binary Constraint

  • Relates two variables.
  • Example, v1 != v2
  • A binary CSP is the one with only binary constraints, and it can be represented as a constraint graph

Global Constraint

  • Involving an arbitrary number of variables (Note: Not all varaibles should be involved)
  • Example: Alldiff, meaning variables involved should have different values

Constraint Hypergraph

  • Consists of
    1. Ordinary nodes: usually circles, representing variable
    2. Hypernodes: usually square, representing n-ary constraints

Backtracking Search

Improving Backtracking efficiency

Least Constraining Value

choose the variable that rules out the fewest values in the remaining variables

Minimum Remaining Values

choose the variable with the fewest legal values; when encoutering tie, choose the one with most constraints (this will lead to problems, if any, sooner rather than delaying them)

Forward Checking

For each of the unassigned variable, we will keep track of the remaining legal values; if we ever encountered a scenario such that a variable does not have any legal values remained, we stop and backup

It is an early waning system basically

Constraint Propagation and Arc Consistency

Node Consistency

A single variable is called node consistent if all the values in the variable’s domain satisfy the variable’s unary constraints

A network is node consistent if every variable in the network is node-consistent

Arc Consistency

A variable is arc consistent if every value in its domain satisfies the variable’s binary constraints.

More formally, Xi is arc consistent with respect to another variable Xj if for every value in the current domain Di there is some values in the domain Dj that satisfies the binary constraint on the arc (Xi, Xj)

A network is arc consistent if every variable is arc consistent with every other variable

AC-3 algorithm

  • Most popular algorithm to solve arc consistency
