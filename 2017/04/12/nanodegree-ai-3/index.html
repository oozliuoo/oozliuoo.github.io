NanoDegree AI - Intro to Search

Posted on 2017-04-12

Definition of a problem

  • Initial State -> S0
  • Actions (s) -> {a1, a2, a3, …}
  • Result (s, a) -> s’ (a new state)
  • GoalTest(s) -> True|False
  • PathCost (sequence of state, actions and transtions) -> cost value(n)

State

  • Three areas:
    • Frontier
    • Explored
    • Unexplored

General tree search

General Graph Search:

Breadth-First Search

  • Shortest First Search
  • When doing remove.choice(frontier), it will always pick the one nearest to current state first
  • The goal test is passed when we remove a state from frontier
  • Guarantee to find shortest path if there is no/equal path cost

Depth-First Serch

  • Longest First search
  • When doing remove.choice(frontier), it will always pick the one furthest to current state first

Uniform Cost Search:

  • Cheapest First Search
  • When doing remove.choice(frontier), it will always pick the one with lowest total path cost so far first
  • Time wasted in searching every direction in order to locate the goal -> If we have an estimate of the distance from S to G, we could find the goal faster

Greedy Best-First Search:

  • Expand the one has least estimate of distance first
  • It will stick to the correct direction, but will cause problem if there are obstacles. (normally will end up longer path)

A* search

  • Combine with Uniform Cost Search and Greedy Best-First Search: Best estimated total path cost first
  • Always expand the node that has Minimum Value of f = g + h, where
    • g(path) = path cost
    • h(path) = estimated distance from that node to goal
  • Result: Search stratetgy that is the best possible, and finds the shortest path while expanding minimum number of paths possible
  • It is ot guaranteed to find the shortest cost path, as it depends on the heuristic function

Properties of Searches

  • Optimal: is it guaranteed to find the shortest path
    • Breath First Search
    • Uniform Cost Search
  • Completeness: is it guaranteed to find the goal, if there is a goal somewhere (consider infinite tree/graph)
    • Breath First Search
    • Uniform Cost Search

Optimistic Heuristic

  • We say that a heuristic function is optimistic or admissible if it never overestimates distance to goal: h(s) < true cost
  • How to come up with good heuristic function? - Gracefully relax the problem

Problems with search

  • Problem must be
    1. Fully Observable: must be able to see what initial we start out with.
    2. Known: must know the set of available actions to us
    3. Discrete: must be a finite number of actions to choose from
    4. Deterministic: must know the result of taking an action
    5. Static: must be nothing else that can change thet world other than our own action
